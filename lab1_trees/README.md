# Отчёт по лабораторной работе №1. Работа с деревом.
## Автор
Китаева Дарья Максимовна. Группа: М8О-102БВ-25.

## Цель работы
Реализовать программу на языке С для построения и обработки дерева общего вида. Основные функции: добавление узла, тектовая визуализация дерева, удаление, вычисление значения функции проверки дерева на монотонность возрастания ширины (true/false).

## Возможности
Поддержка операций:
- создание дерева (корня);
- добавление узла;
- вывести созданное дерево;
- удалить дерево;
- проверить монотонно ли возрастает ширина.

## Основные функции
- **createtree** - создаёт новое дерево(корень), выделяет под него память с помощью функции malloc. Присутствуют поля data и nodeleft/noderight(которые являются указателями на детей данного узла; по умолчанию они - NULL).
- **printTree** - работает на основе очереди: берём элемент, который присутствует в очереди (корень или узел); выводим уровень, на котором находится данный узел; с помощью цикла for проходимся по детям данного узла(если значение ребёнка != NULL, то добавляем его в очередь, так просматриваем всех детей данного узла); далее таким же образом проходимся по детям(следующим узлам, которые мы добавили в очередь, также добавляя в очередб их детей).
- **insertNode** - берём узел из очереди; проверяем есть ли у него правый и левый дети, если нет одного из них, то на это место добавляем новый узел, если же оба ребёнка уже существовали, то добавляем их в очередь.
- **deleteTree** - проверяем пусто ли дерево уже, с помощью указателя на корень, если нет, то вызываем функцию freeTree, которая освобождает память, занимаемую деревом и после меняем указатель на дерево на NULL.
``` c
void freeTree(Tree* root) {
    if (root == NULL) return;
    freeTree(root->nodeleft);
    freeTree(root->noderight);
    free(root);
}
```
- **treemono** - проверяет монотонность увеличения ширины дерева.

## Функция treemono (вариант №10)
1. проверяем не пусто ли наше дерево, если да, то возвращаем true, иначе переходим к другим шагам;
2. объявляем переменные, которые понадобятся нам далее во время выполнения программы, а также инициализируем очередь, на которой строится наше дерево.
```c
int previous_width = 0; // отвечает за ширину предыдущего уровня
int level = 0; // уровень (корень, дети корня, и тд)
```
3. запускаем бесконечный цикл (который останавливает действие всей функции во время выполнения условий), внутри которого объявляем перменную current_width для определения ширины нынешнего уровня, которая равна размеру очереди;
4. проверяем если уровень != корню, если да, то проверяем current_width <= previous_width, если это тоже верно, то возвращаем значение false и освобождаем память, занимаемую очередью;
5. если условие под п.4 не прошло, то приравниваем previous_width = current_width для того, чтобы далее мы могли перейти на следуюзий уровень, где находятся дети узлов, которые находятся на данной ширине;
6. заходим в цикл for, который идёт от 0 до значения current_width, в нём же объявляем переменную типа Tree, которая забирает значение из очереди (удаляя его из очереди) и после этого внутри цикла анализируем детей этого узла;
7. если значение ребёнка (левого или правого) не равно NULL, то мы добавляем этого ребёнка в очередь, делаем мы этого для того, чтобы в очереди далее остались только дети предыдущих узлов(которые находились на предыдущем уровне), после мы уходим снова в цикл и проверяем, что current_width <= previous_width или нет;
8. увеличиваем значение счётчика level, если не выполнился п.4;
9. после выполнения цикла, описанного в п.3, освобождаем память, занимаемую очередью, а также возвращаем true:
```c
free(q); // освобождение памяти
return true; // завершаем функцию и передаём значение true, тк до этого не было передано значение false
```
