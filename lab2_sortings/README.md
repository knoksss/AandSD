## Основные функции
- **counting_sort** - Производит сортировку подсчётом. Выделяет память под два вспомогательных массива (count и result) с помощью malloc. Присутствуют поля для хранения временных данных и указатели на исходный массив таблицы. По умолчанию все счетчики обнуляются. Вспомогательная функция, нужная для ```counting_sort```:
```c
int compare(PutData* a, PutData* b){
    int str_cmp = strcmp(a->str_part, b->str_part);
    if (str_cmp < 0) return -1;
    if (str_cmp > 0) return 1;
    if (a->int_part < b->int_part) return -1;
    if (a->int_part > b->int_part) return 1;

    return 0;
}
```
- **binary_serch** - Ищет элемент в отсортированной таблице по двум критериям: строке `word` и числу `number`. Сужает границы поиска с помощью переменных `left` и `right`, которые обозначают левую и правую границы текущего диапазона (по умолчанию от 0 до последнего индекса). Внутри цикла вычисляется средний индекс `middle`, и в зависимости от сравнения со строкой и числом границы смещаются. Если элемент найден — возвращает его индекс, иначе возвращает -1.
- **read_file** - Выделяет память под новый элемент PutData и его поля (str_part и data) с помощью malloc. Присутствуют поля для хранения строковой части, целого числа и данных. При ошибках выделения памяти освобождает уже выделенное (по умолчанию NULL).


## Функция counting_sort (сортировка линейным выбором с подсчётом)

1. Проверяем, что таблица не пуста и содержит хотя бы один элемент. Если таблица пуста (размер равен 0), то сортировка не имеет смысла - завершаем функцию.

2. Выделяем динамическую память под два вспомогательных массива:
   - Массив `count` размером `t->size` для подсчёта количества элементов, которые меньше каждого элемента
   - Массив `result` размером `t->size` для временного хранения отсортированных элементов
   
   Если память не удалось выделить, выводим сообщение об ошибке и завершаем функцию.

3. Инициализируем массив `count` нулями. Для этого проходим циклом от 0 до `t->size - 1` и устанавливаем каждому элементу значение 0.

4. Запускаем двойной цикл для подсчёта позиций каждого элемента:
   - Внешний цикл проходит по всем элементам таблицы (индекс `i` от 0 до `t->size - 1`)
   - Внутренний цикл также проходит по всем элементам таблицы (индекс `j` от 0 до `t->size - 1`)
   
   Для каждой пары элементов `i` и `j`:
   - Проверяем, что это разные элементы (`i != j`)
   - Сравниваем элементы с помощью функции `compare(t->array[i], t->array[j])`
   - Если элемент `i` больше элемента `j` (результат сравнения > 0), то увеличиваем `count[i]` на 1
   
   Таким образом, после завершения циклов в `count[i]` будет храниться количество элементов, которые меньше элемента `i`, а значит, и его позиция в отсортированном массиве.

5. Расставляем элементы по правильным позициям в массиве `result`:
   - Проходим циклом от 0 до `t->size - 1`
   - Для каждого элемента `i` получаем его позицию `pos = count[i]`
   - Помещаем указатель на элемент `t->array[i]` в массив `result` на позицию `pos`
   
   Благодаря этому шагу все элементы оказываются в массиве `result` в отсортированном порядке.

6. Копируем отсортированные элементы обратно в исходную таблицу:
   - Проходим циклом от 0 до `t->size - 1`
   - Присваиваем `t->array[i] = result[i]`
   
   Теперь исходный массив `t->array` содержит элементы в отсортированном порядке.

7. Освобождаем динамическую память, выделенную под вспомогательные массивы:
   - `free(count)` - освобождаем массив счётчиков
   - `free(result)` - освобождаем временный массив

8. Функция завершается. Таблица отсортирована по комбинированному ключу (сначала по строке, затем по числу).
