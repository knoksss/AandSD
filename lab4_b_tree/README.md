# Лабораторная работа №4: Сбалансированные деревья

## Автор
Китаева Дарья Максимовна. Группа: М8О-102БВ-25.

## Цель работы
Построить упорядоченное дерево для хранения и поиска элементов с уникальными строковыми ключами и вещественными значениями. Ключ – непустая последовательность латинских букв не более чем из 6 символов. Ключи упорядочиваются лексикографически. 

## Возможности
- Реализовано **B-дерево порядка 2**
- Поддерживаются операции:
  - 1: **добавление** пары `<key, value>` (ключ уникальный)
  - 2: **удаление** по ключу
  - 3: **печать** дерева
  - 4: **поиск** значения по ключу
- Поддерживается ввод команд из **текстового файла** в формате:
  - `<номер операции> [<ключ>] [<значение>]`

## Описание (как устроено B-дерево)
B-дерево степени \(t\) хранит в каждом узле от \(t-1\) до \(2t-1\) ключей (кроме корня) и имеет от \(t\) до \(2t\) детей.  
При **вставке** если ребёнок переполнен — он **делится (split)**, средний ключ поднимается в родителя.  
При **удалении** мы заранее следим, чтобы при спуске в ребёнка в нём было минимум \(t\) ключей: если меньше — делаем **заимствование** у соседа или **слияние (merge)**.

## Сборка
Выполните в терминале:
```sh
make all
```
Будут собраны main.exe и tests.exe.

## Запуск
Для запуска основной программы:
```sh
./main
```
Можно передать файл с командами:
```sh
./main commands.txt
```

## Тестирование
Для запуска тестов:
```sh
./tests
```

## Очистка
Для удаления всех скомпилированных файлов и тестовых данных:
```sh
make clean
```
